# Security Policy

This policy is governed by [SZL Doctrine v11](https://github.com/szl-holdings/.github/blob/main/DOCTRINE_V11.md),
including the No-Hallucinations / No-Fake-Green and Every-Claim-Citable clauses.
We do not publish security claims we cannot back with a verifiable source.

## Reporting a vulnerability

Report security vulnerabilities to **security@szlholdings.com**.
If you do not receive a response, use **stephen@szlholdings.com** as a fallback.

Please do NOT open a public issue for security reports.

## Scope

- Code on the current default branch (`main`) of this repository (szl-holdings/killinchu).
- Released, signed artifacts published from this repository.

Out of scope: third-party dependencies (report upstream), and forks outside the
szl-holdings organization.

## Response

We acknowledge valid reports and coordinate disclosure timelines with the
reporter. Fixes land on `main` with SLSA-signed provenance.
